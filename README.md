<div style="background-color: #0D1117; border: 1px solid #30363D; border-radius: 8px; padding: 20px; margin-bottom: 20px; font-family: system-ui, sans-serif; color: #C9D1D9;">
  <h2 style="color: #FF8800; border-bottom: 1px solid #21262D; padding-bottom: 10px; margin-top: 0;">📖 Overview</h2>
  <p>Enforces Spec-Driven Development (SDD) protocols across complex agentic software engineering tasks. Instead of writing code immediately, the agent follows a disciplined, multi-step specification, planning, and task-decomposition process to maximize correctness and align architecture before execution begins.</p>
</div>

<div style="background-color: #0D1117; border: 1px solid #30363D; border-radius: 8px; padding: 20px; margin-bottom: 20px; font-family: system-ui, sans-serif; color: #C9D1D9;">
  <h2 style="color: #FF8800; border-bottom: 1px solid #21262D; padding-bottom: 10px; margin-top: 0;">🔄 SDD Phase Breakdown</h2>
  
  <ol style="padding-left: 20px; margin-bottom: 0;">
    <li style="margin-bottom: 10px;">
      <strong style="color: #FF8800;">Constitution Alignment (<code>/speckit.constitution</code>):</strong> Verify architectural conventions and technical guardrails in <code>.speckit/constitution.md</code> before writing any files.
    </li>
    <li style="margin-bottom: 10px;">
      <strong style="color: #FF8800;">Specification First (<code>/speckit.specify</code>):</strong> Draft target behaviors, user stories, and UI/UX flows in <code>.speckit/specs/</code> prior to defining tech stacks or writing code.
    </li>
    <li style="margin-bottom: 10px;">
      <strong style="color: #FF8800;">Ambiguity Resolution (<code>/speckit.clarify</code>):</strong> Formulate precise questions to clarify edge cases, security requirements, and missing constraints with the user.
    </li>
    <li style="margin-bottom: 10px;">
      <strong style="color: #FF8800;">Technical Planning (<code>/speckit.plan</code>):</strong> Map out exact file layouts, schemas, API contracts, and step-by-step implementations in <code>.speckit/plans/</code>.
    </li>
    <li style="margin-bottom: 0px;">
      <strong style="color: #FF8800;">Task Decomposition & Execution (<code>/speckit.tasks</code> -> <code>/speckit.implement</code>):</strong> Break the plan into tiny tasks, implementing and testing incrementally (TDD).
    </li>
  </ol>
</div>
